**版本:** 2.0 | **目标:** 精简架构，分层处理

## 1. 概述

### 1.1 设计理念

TDR (Time Data Ring) 是一种**定长环形存储结构**，专为边缘计算和时序状态记录设计。

**核心思想**:
- **空间等于时间**: 存储空间视为首尾相连的时间环，物理位置通过同余映射到时间点
- **固定大小**: 创建时即分配全部空间，生命周期内体积恒定，无需动态分配或 GC
- **近期窗口**: 仅关心最近一段时间的数据，过期数据自动被新数据覆盖

### 1.2 应用场景

- **边缘传感器存储**: 温度、湿度、压力等时序数据（支持逻辑值、整数或映射后的浮点数）
- **设备状态记录**: CPU/内存使用率、状态位图，在极低资源占用下持续运行
- **数据采样**: 访问量、性能指标的定期采集，保存最近样本

### 1.3 技术选型

- **内存映射 (mmap)**: 操作系统级的内存-文件映射，确保 O(1) 访问速度
- **位压缩存储**: 支持 n ∈ {1,2,4,8,16,32,64,128}，最大化空间利用率
- **小端序**: 所有多字节整数采用 Little-Endian，兼容主流处理器

## 3. 头部定义

固定 512 字节，小端序。

| 偏移 (Hex) | 长度 | C 类型 | 字段 | 约束 |
|:---|:---|:---|:---|:---|
| `0x00` | 4 | `char[4]` | **Magic** | 固定为 `{'T','D','R',0}` |
| `0x04` | 4 | `uint32_t` | **Version** | 固定为 `1` |
| `0x08` | 4 | `uint32_t` | **HeaderSize** | 固定为 `512` |
| `0x0C` | 4 | `uint32_t` | **Length (L)** | 缓冲区容量，`L > 0` |
| `0x10` | 4 | `uint32_t` | **Step** | 时间步长（ms），`Step > 0` |
| `0x14` | 1 | `uint8_t` | **BitWidth (n)** | `n ∈ {1,2,4,8,16,32,64,128}` |
| `0x15` | 3 | `uint8_t[3]` | **Reserved** | 填充 `0x00` |
| `0x18` | 8 | `uint64_t` | **BeginTS** | 起始时间戳，`0` 表示未写入 |
| `0x20` | 8 | `uint64_t` | **LastTS** | 最新时间戳，`0` 表示未写入 |
| `0x28` | 472 | `char[472]` | **Remarks** | UTF-8 字符串，不足补 `\0` |

**Struct 格式**: `<4sIIIIIB3xQQ472s` (共 512 字节)

## 4. 读取流程

```
输入: timestamp

1. 有效性校验 (任一满足则返回 None):
   - timestamp < BeginTS          (早于首次写入)
   - timestamp > LastTS           (未来时间)
   - timestamp <= LastTS - L×Step (已被覆盖)

2. 计算物理位置:
   Tick  = floor((timestamp + Step/2) / Step)
   Index = Tick % L
   ByteOffset = 512 + (Index * n) // 8
   BitShift   = (Index * n) % 8

3. 从 ByteOffset 处读取 n 位数据
```

---

## 5. 写入流程

```
输入: timestamp, value, pad_value

1. 首环记录:
   若 BeginTS == 0:
       BeginTS = LastTS = timestamp

2. Gap 检测:
   Gap = timestamp - LastTS
   
   情况 A: 超环间断 (Gap >= L × Step)
       → 重置 BeginTS = LastTS = timestamp
       → 旧数据自动失效，无需清空
   
   情况 B: 环内间断 (Step < Gap < L × Step)
       → 计算填充范围 [curr_idx+1, new_idx-1]
       → 调用间隙填充（见 6.3 节分层架构）
   
   情况 C: 正常追加 (Gap == Step)
       → 无需填充

3. 写入当前值到 timestamp 对应位置

4. 更新 LastTS = timestamp
```

## 6. 核心机制

### 6.1 索引计算

```
Tick       = floor((Timestamp + Step/2) / Step)    # 四舍五入
Index      = Tick % L                               # 环形索引
BitOffset  = Index × n
ByteOffset = 512 + BitOffset // 8
BitShift   = BitOffset % 8
```

### 6.2 为什么只有"增"，没有"改"和"删"

TDR 是一种**时序追加型存储**，其设计哲学决定了三种操作的取舍：

| 操作 | 支持状态 | 原因 |
|:---|:---|:---|
| **增 (Append)** | ✅ 核心操作 | 时间单调递增，新数据按时间顺序写入 |
| **改 (Update)** | ❌ 不支持 | 历史数据是**不可变的事实**，修改会破坏时序完整性 |
| **删 (Delete)** | ❌ 不支持 | 通过**环形覆盖**实现隐式删除，无需主动清理 |

**"删"的替代方案——环形覆盖**:
- 当写入的时间戳超出 L×Step 窗口时，旧数据被新数据自动覆盖
- 读取时通过 `timestamp <= LastTS - L×Step` 判定为已失效
- 超环间断时直接重置时间戳，旧数据立即可达性失效

**"改"的替代方案——补偿记录**:
- 若数据有误，应写入一条新的补偿记录（带新的时间戳）
- 应用层根据时间戳顺序解析，以最新记录为准

这种设计的好处：
1. **实现极简**: 无需实现复杂的更新/删除逻辑
2. **性能最优**: 纯追加写，无随机修改
3. **可追溯性**: 完整的历史记录（在窗口期内）

### 6.3 区间(间断)填充分层架构

```
L1 业务层 ────→ TDR.write() 判定 Gap，计算索引范围
      ↓
L2 解环层 ────→ 判断是否跨越环尾，分段处理
      ↓
L3 分流层 ────→ n≥8 直接字节填充；n<8 转入处理边界字节对齐问题
      ↓
L4 字节对齐三段模型 ──→ 头-体-尾分解，按位处理头尾，体部转入字节填充
      ↓
L5 字节填充 ──→ 构造填充模式，对64KB取模部分构造切片缓冲buffer填充，对64KB簇使用分块切片缓冲blockBuffer循环填充
```

#### 6.3.1 各层职责

| 层级 | 输入 | 输出 | 关键操作 |
|:---|:---|:---|:---|
| **L1** | `timestamp`, `value` | `start_idx`, `end_idx` | Gap 检测，超环判定 |
| **L2** | 索引范围 | 1-2 个连续子范围 | 环形跨越拆分 |
| **L3** | 子范围，`n` | 字节范围 或 位范围 | n≥8 直接字节；n<8 用字节对齐三段模型 |
| **L4** | 位范围，`pattern` | 头/体/尾边界 | 计算边界，头尾位运算(RMW)，体部调 L5 |
| **L5** | 字节地址，`size`，`pattern` | mmap 写入 | 64KB分块阈值判断，blockBuffer复用，切片缓冲构造 |

#### 6.3.2 术语定义

| 对象 | 大小 | 说明 | 示例 |
|:---|:---|:---|:---|
| **位** | 1bit | 最小存储单元 | `0b0` |
| **字节** | 8位=1B | 基本寻址单元 | `0xff` |
| **位宽(n)** | uint8 | 2的幂次: 1,2,4,8,16,32,64,128 | `2` |
| **值** | n位 | 实际存储的数据 | `0b01` |
| **填充值** | n位 | 用于区间填充的值 | `0b11` |
| **填充模式** | max(n,8)位 | 将n位值重复填充到8位 | `0xff` |
| **切片缓冲buffer** | 64B~64KB | 将填充模式重复构造的临时bytes | - |
| **分块切片缓冲blockBuffer** | 64KB | 预分配的64KB填充块，可复用 | - |
| **64B逐值阈值** | 512位=64B | 小于此阈值使用逐值写入 | - |
| **64KB分块阈值** | 65536字节 | 大于此阈值使用分块循环填充 | - |

#### 6.3.3 区间填充策略表

| 区间长度\数据位宽 | n<8（字节对齐问题） | n≥8 |
|:---|:---|:---|
| **<64B** | 逐值写入 | 逐值写入 |
| **64B ~ 64KB** | 头体尾字节对齐+体部切片写入 | 切片写入 |
| **≥64KB** | 头体尾字节对齐+体部切片&分块切片写入 | 切片写入+分块切片写入 |

#### 6.3.4 字节对齐三段模型（n < 8）

将位范围 `[bit_start, bit_end]` 分解为头-体-尾：

```
示例 (n=2, 填充索引 3-10，即 bit 6-21):

  Byte0       Byte1       Byte2
  [xx 11 1111] [1111 1111] [11 xx xxxx]
   ↑  ↑         ↑          ↑  ↑
   │  │         │          │  │
   │  └─────────┴──────────┘  │
   │        体部(16位)        │
   │                          │
头部(6位)                  尾部(2位)

处理策略:
- 头部/尾部: Read-Modify-Write，保护同字节其他位
- 体部: 字节对齐，调用 L5 批量填充
```

#### 6.3.5 字节填充优化（L5）

```python
def byte_fill(mmap, start, size, pattern):
    """
    L5: 字节填充.
    
    构造填充模式，对64KB取模部分构造切片缓冲buffer填充，
    对64KB簇使用分块切片缓冲blockBuffer循环填充.
    """
    BLOCK = 64 * 1024  # 64KB分块阈值
    offset = start
    remaining = size
    
    # 处理 64KB 整数倍: 使用分块切片缓冲 blockBuffer 循环填充
    while remaining >= BLOCK:
        mmap[offset:offset+BLOCK] = _block_buffer  # 预分配，复用
        offset += BLOCK
        remaining -= BLOCK
    
    # 处理剩余: 构造切片缓冲buffer填充
    if remaining > 0:
        mmap[offset:offset+remaining] = bytes([pattern]) * remaining
```

#### 6.3.6 填充策略选择说明

| 填充大小 | 策略 | 原因 |
|:---|:---|:---|
| < 512 位 (64 B) | 逐值写入 | 小于64B逐值阈值，避免构造buffer的开销 |
| 64 B ~ 64 KB | 切片缓冲buffer | 构造临时切片缓冲，平衡内存分配和复制效率 |
| ≥ 64 KB | 分块切片缓冲blockBuffer | 达到64KB分块阈值，使用预分配块复用减少内存分配 |

## 7. 关键算法

### 7.1 模式字节生成

```python
def make_pattern(n: int, v: int) -> int:
    """将 n 位值 v 重复填充到 8 位字节"""
    mask = (1 << n) - 1
    pattern = 0
    for i in range(8 // n):
        pattern |= (v & mask) << (i * n)
    return pattern

# n=2, v=3(0b11)  → 0b11111111 = 0xFF
# n=4, v=5(0b0101) → 0b01010101 = 0x55
```

### 7.2 部分字节填充

```python
def fill_partial_byte(data, addr, shift, bits, pattern):
    """填充字节的指定范围 [shift, shift+bits)"""
    old_val = data[addr]
    fill_mask = (1 << bits) - 1
    fill_val = (pattern >> shift) & fill_mask
    clear_mask = ~(fill_mask << shift) & 0xFF
    data[addr] = (old_val & clear_mask) | (fill_val << shift)
```

## 10. 易错点提示

1. **模式字节 vs 原始值**: n<8 时必须重复填充 pattern，不能直接写 value
2. **字节序**: 所有多字节整数使用小端序
3. **边界计算**: `bit_end = (end_idx + 1) * n - 1`，不是 `end_idx * n`
4. **环形跨越**: `start_idx > end_idx` 时表示跨越，需拆分为 [start, L-1] 和 [0, end]
5. **头/尾保护**: 部分字节填充必须 Read-Modify-Write，不能覆盖整个字节
6. **超环情况**: Gap >= L×Step 时无需清空数据区，直接重置时间戳即可
